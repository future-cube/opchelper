# OPC Helper（一人公司助手）网站运营方案（v1）

> 面向：独立开发者/自由职业者/小型工作室（1–3 人），希望用“系统化方法 + 可执行教程 + 资源导航 + 模板下载”把一人公司跑起来，并持续增长。

## 1. 定位与一句话价值主张

**定位：** 一人公司（OPC）的“经营操作系统”（Operating System）+ “能力学校”：把零散经验沉淀成可复用的 SOP、模板、清单与路线图。

**一句话：** 用一套“可复制的 SOP + 模板 + 工具”，帮助一人公司把业务从“靠灵感”升级到“靠系统”。

## 2. 核心用户与需求画像

### 2.1 目标人群

- A 类：自由职业者（设计/开发/写作/咨询），项目制收入波动大。
- B 类：独立开发者（产品/工具/插件/课程），缺少稳定增长与反馈闭环。
- C 类：小工作室（1–3 人），协作成本与交付管理缺工具。

### 2.2 核心痛点（按优先级）

1. 获客不稳定：线索来源单一、转化话术缺、缺少内容资产。
2. 交付不可控：需求变更多、范围漂移、沟通成本高。
3. 现金流波动：报价/合同/催款流程不标准，回款周期长。
4. 精力分散：同时做销售/交付/运营，缺少节奏与复盘机制。

## 3. 产品形态（网站模块）

### 3.1 内容资产（SEO / 可分享）

站点要从“工具站”升级为“学习 + 导航 + 下载 + 落地工具”的组合：

- **Guides（学习）**：能力教程/方法论/清单/模板（SEO 主力，中文优先 + 英文摘要）
- **Playbooks（手册）**：按行业/业务类型的“从 0 到 1 SOP”（系统化、可持续更新）
- **Directory（导航）**：大模型供应商/中转平台/工具/站点（强调“适合谁/不适合谁”）
- **Downloads（下载）**：可直接下载的模板包/清单包（提高复访与转发）
- **Tools（工具）**：浏览器内工具（v1 不需要密钥），把方法落地成可复制产出

### 3.2 工具型页面（高转化）

- **模板生成器**（浏览器内生成，可复制）：项目简报、报价说明、合同要点、催款邮件、周报/复盘等。
- **清单与自检**：项目启动清单、交付前检查、风险预案清单。
- **成本与定价计算器**（后续）：时薪/日薪换算、保底收入目标拆解。

### 3.3 内容主航道（v1 先跑通 4 条）

建议用“频道/航道”的方式运营（每条航道都有 Guides + Directory + Downloads）：

1. **AI / LLM 工具导航与教程**：OpenClaw、Codex CLI、Claude Code、Cursor、opencode、主流模型供应商与中转平台
2. **自媒体变现**：内容 → 转化 → 产品化（newsletter、赞助、付费社群、数字产品）
3. **服务/咨询交付与回款**：范围控制、交付节奏、验收与回款系统
4. **虚拟资料/模板销售**：选题、验证、定价、交付、复购

### 3.3 订阅与增长（可持续）

- **免费**：基础模板 + 操作手册部分章节 + 每周邮件（可选）。
- **付费（后续迭代）**：
  - Pro：高级模板、可下载包、版本更新、案例库深度内容。
  - Studio：团队协作版（1–3 人）、项目看板、客户门户（更后）。

## 4. 获客渠道与内容策略

### 4.1 渠道组合（从低成本开始）

- **SEO（主）**：关键词围绕“报价/合同/催款/需求变更/自由职业/一人公司”。
- **社媒分发（辅）**：小红书/知乎/微信公众号/即刻/掘金（按你常用平台选择 2 个）。
- **合作/互推**：与设计师、开发者社群、工具类博主互推。
- **产品社区**：Product Hunt（若有英文版）、V2EX/少数派等。

### 4.2 内容框架（每篇都能指向工具）

- 问题型：如“如何写一封不尴尬的催款邮件？”
- 清单型：如“交付前 10 项风险检查”
- 模板型：如“需求变更说明模板（可复制）”
- 案例型：如“客户临时加需求，我如何把它变成加价机会”
- 导航型：如“2026 你该怎么选模型供应商/中转平台/本地工具栈？”

### 4.3 内容节奏（可执行）

- 每周 2 篇短内容（500–800 字）：问题/清单/模板
- 每两周 1 篇长文（1500–3000 字）：系统性方法 + 案例
- 每月 1 次“工具更新”：新增 2–4 个模板/生成器字段

### 4.4 编辑流程（强制落地）

v1 不做后台管理，但要把“更新流程”写进习惯：
1. 研究：先记到 `docs/research/YYYY-MM-DD-<topic>.md`（官方链接 + 变化点）
2. 写作：写进 `code/content/zh/**`（中文主文，可执行）
3. 摘要：补齐 `code/content/en/**`（英文摘要，不可空）
4. 内链：至少链接 2 个站内页面（Guides/Playbooks/Directory/Downloads）
5. 发布：`pnpm -C code generate` 验证静态站能生成，再 push

## 5. 转化设计（从访客到留资）

### 5.1 首屏与 CTA

- 首屏：一句话价值 + “立即使用模板生成器”按钮 + “下载免费模板包”备选入口。
- 关键页面：工具页置顶“复制结果”与“保存为 Notion/Markdown”。

### 5.2 留资机制（阶段性）

v1：不强制登录，先提升分享与口碑。
v2：可选邮箱订阅（更新提醒、每周一封）。
v3：付费墙（高级模板/下载包/历史版本）。

## 6. 指标体系（最小可用数据）

### 6.1 北极星指标（NSM）

- **每周活跃工具使用人数**（工具页 UV → 复制次数/生成次数）

### 6.2 关键漏斗

- 访问 → 阅读/导航点击 → 下载/工具使用 → 复访 → 订阅 → 付费（后续）

### 6.3 早期目标（上线后 4 周）

- 每周新增内容 ≥ 2 篇
- Directory 条目新增 ≥ 10（含官方链接与标签）
- 工具页每周使用人数 ≥ 100（或 Downloads 每周下载 ≥ 50）
- 邮箱订阅转化率 ≥ 1–3%（上线订阅后）

## 7. 运营工作流（每周固定节奏）

- 周一：选题 + 数据回看（上周工具使用/文章表现）
- 周二：发布短文 + 同步工具页内链
- 周四：发布短文（模板/清单）
- 周五：新增/优化 1 个模板（字段更贴近真实场景）
- 周末：写长文或案例拆解（两周一次）

## 8. 版本路线图（与开发计划对齐）

### v1（本次）

- 上线一个高质量官网 + 3–5 个工具模板生成器
- 形成内容框架与内链结构（工具 <-> 手册）

### v1.1

- 增加“定价/回款”模块与更多模板
- 增加基础分析（按钮点击/复制次数）

### v2

- 邮箱订阅、模板包下载、基础用户分层
