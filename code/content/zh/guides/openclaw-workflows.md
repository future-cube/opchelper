> 说明：这篇不是“功能列表”，而是把一人公司最常见的 3 件事做成闭环。  
> 工具会变，但闭环不变：**输入清晰 → 输出可用 → 可复盘 → 可复用**。

## 0）使用前提（默认只读）

建议你先把安全边界定好：
- `/zh/guides/openclaw-safety`

## 1）闭环 A：内容生产（选题 → 草稿 → 发布清单 → 复盘）

### 目标

把“靠灵感发内容”变成“按栏目与 CTA 产出”。

### 输入（你提供）

- 选题一句话（目标读者是谁、解决什么问题）
- 3–5 个参考来源（优先官方文档/权威文章）
- 输出平台（公众号/知乎/小红书/博客）

### 输出（你要拿到）

- 一份结构化大纲（含小标题、要点）
- 一份草稿（可直接润色发布）
- 一份发布清单（标题、封面、CTA、链接、标签）
- 一段复盘提示（本次要验证的指标）

### 最小 SOP（可复制）

1) 让它先“读/总结”来源（不写入项目，只输出到 `outputs/`）  
2) 再让它写大纲（严格按你指定的栏目结构）  
3) 再写草稿（明确“不要虚构数据、不要编造引用”）  
4) 最后生成发布清单与复盘问题

站内可配合使用：
- 下载内容日历模板：`/zh/downloads/content-calendar`
- 自媒体漏斗地图：`/zh/guides/newsletter-funnel`

## 2）闭环 B：线索跟进（线索 → 触达 → 跟进节奏 → 复盘）

### 目标

把“偶尔聊聊客户”变成“有节奏的跟进系统”。

### 输入

- 线索来源（平台/群/转介绍）
- 对方是谁（角色/行业/痛点）
- 你提供什么（服务范围 + 交付物）

### 输出

- 3 条触达消息（不同语气/场景）
- 7 天跟进节奏（第 1/3/7 天发什么）
- 一份“未成交复盘表”（为什么没成、下次怎么改）

搭配下载：
- 赞助/合作外联模板：`/zh/downloads/sponsorship-outreach`

## 3）闭环 C：交付与回款（简报 → 验收 → 变更 → 催款）

### 目标

减少范围漂移与无休止返工，让“交付可验收、可结款”。

### 输入

- 项目目标、范围、时间、交付物
- 风险点（时间、依赖、资料）
- 付款条款（分期/里程碑）

### 输出

- 项目简报（SOW/Brief）
- 验收清单（可执行）
- 变更说明模板（Change Request）
- 催款邮件（不同语气）

站内直接可用：
- 项目简报模板：`/zh/downloads/project-brief-template`
- 验收清单：`/zh/downloads/acceptance-checklist`
- 需求变更模板：`/zh/downloads/change-request-template`
- 催款邮件：`/zh/downloads/payment-reminder-email`

## 4）怎么把闭环变成“可复用资产”

一人公司最关键的是沉淀资产：

- 每跑通一个闭环，就写成一篇 Guide（步骤 + 模板 + 注意事项）
- 把模板文件放进 Downloads（可直接下载）
- 把工具/供应商放进 Directory（官方链接 + 适合谁）

你可以把输出直接贡献到本仓库结构：
- Guides：`code/content/zh/guides/*.md`
- Downloads：`code/content/zh/downloads/*.md` + `code/public/downloads/**`
- Directory：`code/content/directory/*.json`
