> 目标：让你把 Codex 当成“可控的同事”，而不是“不可控的魔法”。  
> 核心原则：先读懂 → 再小步改动 → 再逐步自动化。

## 0）适合做什么 / 不适合做什么

适合：
- 读代码：理解目录结构、关键入口、调用链
- 改代码：小步 PR（修 bug、重构、补文档）
- 批量任务：批量重命名、生成清单、整理内容

谨慎：
- 未设置边界就让它大范围改动
- 直接触碰密钥/生产配置/个人资料目录

## 1）官方入口（以官方为准）

- 帮助中心：`https://help.openai.com/en/collections/10467008-codex`
- 产品页：`https://openai.com/codex`
- 开源仓库（如果你使用开源 CLI）：`https://github.com/openai/codex`

## 2）先加“护栏”（强烈建议）

### A. 明确不可触碰路径

建议至少包含：
- `.env`、私钥、证书
- 构建产物（`.output/`、`dist/`、`.nuxt/`）
- 大文件与下载目录

### B. 用最小权限开始

第一次建议：只读 → 给计划 → 你确认后再写入/执行。

### C. 小步可回滚

每次让它输出：
- 分步骤计划（3–8 步）
- 每步改动的 diff
- 验证方式（命令 + 预期）

## 3）一人公司高频场景（建议从一个闭环开始）

- 工程闭环：issue → 复现 → 修复 → 测试 → 发布
- 内容闭环：选题 → 资料 → 大纲 → 草稿 → 发布清单
- 交付闭环：简报 → 里程碑 → 变更 → 催款
