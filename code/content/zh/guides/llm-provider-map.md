## 1）先把问题说清楚：你需要的是“模型能力”还是“交付能力”

对一人公司而言，真正重要的是：
- 你能否稳定交付（延迟、限流、可用性）
- 成本能否控住（每次调用成本、失败重试、缓存）
- 法务与合规能否说清楚（地区、数据处理、发票）

## 2）两条路：直连供应商 vs 聚合/中转平台

### 直连供应商
优点：稳定、能力完整、官方文档与支持更直接  
缺点：多家对接成本高、价格/额度/区域差异大

### 聚合/中转平台
优点：一个 API 接多家、路由/容灾更方便  
缺点：需要额外信任与账务处理；某些能力可能不完整

## 3）实用决策流程（建议）

1. 先选 1 个主力供应商（直连）跑通业务
2. 再接入 1 个聚合平台做备份与比价
3. 规模上来后再考虑：自建网关/缓存/观测
