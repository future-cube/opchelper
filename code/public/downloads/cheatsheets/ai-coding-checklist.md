# AI 编程执行清单（适用于 Codex / Claude Code / Cursor / OpenCode / OpenClaw）

> 目标：让你把 AI 工具当成“可控同事”，每次都能 **可验证、可回滚、可复用**。

## 0. 启动前（一次性设置）

- [ ] 项目根目录有规则文件（例如 `AGENTS.md` / `CONTRIBUTING.md`）
- [ ] 明确禁止读取：`.env`、证书/密钥、客户资料、大文件、构建产物
- [ ] 有最小验证命令（build/test/lint）并写进规则

推荐忽略清单（根据项目调整）：

```text
.env
**/*.pem
**/*.key
node_modules/
dist/
.nuxt/
.output/
coverage/
```

## 1. 每次任务的输入（你必须给清楚）

- [ ] 目标（1 句话）：我要达成什么？
- [ ] 约束（3 条以内）：不要改哪里？不要做什么？
- [ ] 验收标准：什么叫完成？怎么验证？
- [ ] 交付物：计划 / diff / 文档 / 下载文件？

可复制提示词：

```text
目标：……
约束：只做最小改动；不要触碰 secrets；不要引入新依赖（除非必要）。
交付物：计划（分步骤）+ 最小 diff + 验证命令（含预期结果）。
```

## 2. 执行节奏（稳定的四步循环）

1) 读懂：先总结现状与关键入口  
2) 计划：3–6 步最小计划（你确认）  
3) 执行：一次只做 1 步，输出最小 diff  
4) 验证：给出命令 + 预期结果 + 风险点

## 3. 提交与回滚（强制可撤销）

- [ ] 每个小任务结束就 commit（信息清晰）
- [ ] 不确定就先不合并，先做第二轮 review
- [ ] 输出写到 `outputs/` 或独立目录，验证通过再迁回主仓库

## 4. 复用（把一次性变成资产）

每跑通一次流程，就沉淀一次：
- 写成 Guide（教程/SOP）
- 把模板放进 Downloads（可下载文件）
- 把工具入口收录到 Directory（官方链接 + 适合谁）
